# توثيق واجهة برمجة التطبيقات (API Documentation)
نظام أتمتة WhatsApp Web

يوفر هذا النظام مجموعة من الواجهات البرمجية (APIs) للتحكم الكامل في حسابات واتساب، وإرسال الرسائل (فردية وجماعية)، وإدارة الردود الآلية.

---

## 1. إدارة الحسابات (Accounts)

الرابط الأساسي: `/api/accounts`

### إنشاء حساب جديد
- **الطريقة:** `POST`
- **المسار:** `/api/accounts/`
- **الوصف:** يقوم بإنشاء حساب جديد ويبدأ عملية التهيئة.
- **جسم الطلب (Body):**
  ```json
  {
    "name": "اسم الحساب (مثال: حساب خدمة العملاء)"
  }
  ```

### جلب جميع الحسابات
- **الطريقة:** `GET`
- **المسار:** `/api/accounts/`
- **الوصف:** يجلب قائمة بكل الحسابات المسجلة وحالتها الحالية (متصل، غير متصل، إلخ).

### جلب رمز الاستجابة السريعة (QR Code) لربط الحساب
- **الطريقة:** `GET`
- **المسار:** `/api/accounts/:id/qr`
- **الوصف:** يجلب رمز الـ QR الذي يجب مسحه باستخدام تطبيق واتساب بالهاتف لربط الحساب بالنظام.

### فحص حالة الحساب
- **الطريقة:** `GET`
- **المسار:** `/api/accounts/:id/status`
- **الوصف:** يعرض الحالة اللحظية للحساب (هل هو متصل أم لا).

### فصل الحساب أو حذفه
- **فصل الحساب (Disconnect):** `POST /api/accounts/:id/disconnect`
- **إعادة تشغيل الحساب (Restart):** `POST /api/accounts/:id/restart`
- **حذف الحساب نهائياً (Delete):** `DELETE /api/accounts/:id`

## 2. إرسال واستقبال الرسائل (Messages)

الرابط الأساسي: `/api/messages`

### إرسال رسالة فردية (مباشرة)
- **الطريقة:** `POST`
- **المسار:** `/api/messages/send`
- **الوصف:** يرسل رسالة نصية أو ملف وسائط إلى رقم محدد باستخدام حساب معين.
- **جسم الطلب (Body):**
  ```json
  {
    "accountId": "معرف الحساب المرسل (ID)",
    "to": "201012345678",
    "message": "نص الرسالة هنا",
    "type": "text", // أو 'image' في حال إرسال ملف
    "mediaPath": "/uploads/filename.jpg" // اختياري (يُستخدم مع الملفات)
  }
  ```

### الإرسال الجماعي (Bulk Send)
- **الطريقة:** `POST`
- **المسار:** `/api/messages/bulk`
- **الوصف:** يضيف مجموعة من الرسائل إلى "طابور الإرسال" (Queue) ليتم إرسالها تدريجياً لتجنب الحظر.
- **جسم الطلب (Body):**
  ```json
  {
    "accountId": "معرف الحساب المرسل (ID)",
    "messages": [
      { "to": "2010...", "message": "رسالة 1", "mediaPath": "" },
      { "to": "2011...", "message": "رسالة 2", "mediaPath": "" }
    ]
  }
  ```

### جلب سجل الرسائل المرسلة
- **الطريقة:** `GET`
- **المسار:** `/api/messages/`
- **الخصائص (Query Parameters):**
  - `accountId`: للفلترة بحساب معين.
  - `status`: للفلترة بالحالة (`SENT`, `PENDING`, `FAILED`).
  - `limit`: عدد الرسائل في الصفحة.
  - `page`: رقم الصفحة.

### استقبال الرسائل (Receiving Messages)
- **الوصف:** لا يحتاج النظام إلى نقطة نهاية (Endpoint) لاستقبال الرسائل، بل يقوم محرك الواتساب (`WhatsAppEngine`) في الخلفية **بالاستماع التلقائي** للرسائل الواردة. عندما يتلقى النظام رسالة جديدة، يقوم بحفظها وتمريرها لنظام "الرد الآلي" (Automation Rules) ليرد عليها تلقائياً إذا كانت تطابق إحدى القواعد المعينة.

---

## 3. الرد الآلي (Automation Rules)

الرابط الأساسي: `/api/automation`

### إضافة قاعدة رد آلي جديدة
- **الطريقة:** `POST`
- **المسار:** `/api/automation/`
- **الوصف:** يقوم بإنشاء قاعدة جديدة للرد التلقائي على الرسائل الواردة بناءً على كلمات مفتاحية.
- **جسم الطلب (Body):**
  ```json
  {
    "accountId": "معرف الحساب (أو يترك فارغاً لتطبيقه على كل الحسابات)",
    "name": "اسم القاعدة (مثال: ترحيب)",
    "triggerType": "exact", // أو 'contains'
    "triggerKeyword": "مرحبا",
    "replyType": "text",
    "replyText": "أهلاً بك! كيف يمكننا مساعدتك؟",
    "isActive": true
  }
  ```

## 4. إحصائيات النظام (Dashboard Stats)

- **الطريقة:** `GET`
- **المسار:** `/api/dashboard/stats`
- **الوصف:** يجلب إحصائيات سريعة عن عدد الحسابات المتصلة، الرسائل المرسلة، وقائمة الانتظار في طابور الإرسال (Queue).

---

## 5. رفع الملفات (Uploads)

- **الطريقة:** `POST`
- **المسار:** `/api/uploads/`
- **الوصف:** يرفع ملف (صورة، مستند، صوت) إلى الخادم (Server) لكي يتم استخدامه لاحقاً في إرسال الرسائل. يتم إرسال الملف باستخدام صيغة `multipart/form-data`.

---

## 6. المصادقة والأمان (Authentication & Security)

جميع المسارات المتعلقة بتسجيل الدخول والمصادقة الثنائية (2FA) وإدارة مفاتيح الـ API.

### إعداد النظام لأول مرة (Setup)
- **الطريقة:** `POST`
- **المسار:** `/api/auth/setup`
- **الوصف:** يقوم بإنشاء حساب المدير (Admin) لأول مرة (تعمل فقط إذا لم يكن هناك أي مستخدمين).
- **جسم الطلب:**
  ```json
  {
    "username": "admin",
    "password": "your_password"
  }
  ```

### تسجيل الدخول (Login)
- **الطريقة:** `POST`
- **المسار:** `/api/auth/login`
- **الوصف:** تسجيل الدخول والحصول على `token`. إذا كانت المصادقة الثنائية مفعلة، ترجع `tempToken`.

### التحقق من المصادقة الثنائية (Verify 2FA)
- **الطريقة:** `POST`
- **المسار:** `/api/auth/verify-2fa`
- **الوصف:** التحقق من رمز الـ OTP لإتمام تسجيل الدخول.

### توليد مفتاح API (Generate API Key)
- **الطريقة:** `POST`
- **المسار:** `/api/auth/api-key/generate`
- **الوصف:** يُنشئ مفتاح API جديد يمكن استخدامه للمصادقة من التطبيقات الخارجية (يُرسل في الهيدر `X-API-Key`).

### إلغاء مفتاح API (Revoke API Key)
- **الطريقة:** `DELETE`
- **المسار:** `/api/auth/api-key`
- **الوصف:** يحذف ويبطل مفتاح الـ API الحالي.
